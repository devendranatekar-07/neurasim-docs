Localization & OTA

Language selection, JSON-based UI translation, remote language file updates, and over-the-air APK installation.

1. Localization

LocalizationManager is a persistent singleton (DestroyedOnSceneChange() → false) that loads a language JSON file on startup and fires a Broker<NewTranslation> event whenever the language changes. In-scene components subscribe to that event to refresh their text.

Assets/Shared/Scripts/LangLocalization/LocalizationManager.cs

Region → Language Mapping

USAenEnglishen.json

The mapping is in ServerToCultureInfo(). Adding a new region requires adding a case there, a corresponding JSON file to StreamingAssets/Localization/, and an entry in lang_meta.json.

The tablet can also override the language by sending a data.lang field (ISO code) over TCP. LoaderController calls LocalizationManager.Instance.LoadLocalizedText() directly when this arrives.

Load & Broadcast Flow

  1. Awake: reads GlobalConfig.serverRegion → calls LoadLocalizedText(culture). If first run after install, copies all JSON files from StreamingAssets/Localization/ to persistentDataPath/Localization/.
  2. LoadLocalizedText: reads {persistentDataPath}/Localization/{iso}.json → deserializes into Dictionary<string, string> → fires Broker<NewTranslation>.Invoke().
  3. Start: calls CheckRemoteLocalization().Forget() to async-check for newer JSON files on the server.
  4. Region change: GlobalConfig.serverRegion.Changed callback → LoadLocalizedText(newCulture).
Language JSON files are always read from persistentDataPath, never directly from StreamingAssets at runtime (Android doesn't support direct file access there). BetterStreamingAssets is used for the first-install copy.

In-Scene Components

LocalizedText

Add to any GameObject with a Text, TextMeshPro, or TextMeshProUGUI component. Set the key field in the Inspector. Subscribes to Broker<NewTranslation> via OnEnable and calls GetValue(key, defaultText) on each language change.

Note: defaultText is captured from the component's current text at OnEnable — set it in the prefab as a fallback.

LocalizedGameObject

Maps ISO codes to child GameObjects — activates the matching one and deactivates the rest. Useful for swapping entire prefab variants per language (e.g. different layout images).

Set updated = true in Inspector if it should react to mid-session language changes; otherwise it only runs in Awake.

LocalizedMaterial

Swaps a Renderer's material based on current language. Same ISO-to-asset mapping pattern as LocalizedGameObject.

Assets/Shared/Scripts/LangLocalization/LocalizedMaterial.cs

Remote Language File Updates (OTA)

On Start, LocalizationManager calls CheckRemoteLocalization(). This fetches {serverUrl}/{hash}/Update/Localization/lang_meta.json and compares each entry's ChangeDateTime with the locally stored meta. Any language whose remote timestamp is newer is downloaded and written to persistentDataPath/Localization/. If the active language was updated, LoadLocalizedText() is called immediately to apply it.

The local meta file is updated after a successful download. No hash/integrity check is applied to language files (unlike the APK updater).

2. OTA Updater

Updater handles over-the-air installation of Android APKs for the main app, stream app, and service app. It uses the device's privileged AndroidHelper.remmedService (a custom Android service) for silent installs, so no user interaction is required.

Assets/Shared/Scripts/Updater/Updater.cs

Check Loop

After Start, if UpdaterEnabled is true, CheckForUpdateLoop(token) runs indefinitely:

  1. Wait 1 s → fetch update_meta JSON from {serverUrl}/{hash}/Update/
  2. For each entry in files_info: compare remote version string against installed APK version using System.Version
  3. If remote > installed and device matches HardwareInfo.DeviceType(): add to download queue
  4. Download each APK with progress bar → silent install via Android service
  5. Fire Broker<UpdatingStateChanged> at start/end; stop TCP servers during install
  6. Wait 4 s → loop again

AppType enum identifies update targets: MAIN_APP, STREAM_APP, SERVICE_APP. The device field in meta matches NeoDeviceType (e.g. Neo3, ANY).

Disabling Updates

UpdaterEnabled returns false when Application.version contains "-no_update" (e.g. 1.2.3-no_update). In the Editor it reads from RemmedMock.Instance.UpdaterEnabled. When disabled, StartChecking() is never called.

TCP servers are stopped during APK installation. The tablet will lose connection momentarily. RunTabletTcpServ() is called in the finally block to restart them after install regardless of success or failure.

3. How-To

Add a Localization Key

  1. Open StreamingAssets/Localization/en.json and pl.json
  2. Add a new { "Key": "MY_KEY", "Val": "My translated text" } entry to the Translation array in each file
  3. Update lang_meta.json — bump the ChangeDateTime for each modified language so remote devices pick up the new file
  4. Redeploy the JSON files to the server under {hash}/Update/Localization/
In Editor, validation runs in LocalizationManager.OnValidate() — it checks that all JSON files exist for every ServerRegion and logs errors for missing keys. Run it by making any change on the LocalizationManager GameObject.

Localize a UI Text

  1. Add LocalizedText component to the GameObject
  2. Set key to the JSON key (e.g. "MY_KEY")
  3. Set the TextMeshPro text to the English fallback — this becomes defaultText if the key is not found
  4. No code needed — the component handles Broker<NewTranslation> subscription automatically

For runtime lookup in code: LocalizationManager.Instance.GetValue("MY_KEY", "fallback") — returns the fallback and logs a warning if the key is missing.
