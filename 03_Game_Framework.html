Game Framework (UGC)

The three-component scaffold every therapy game builds on: controller, parameters, and VFX

Overview

The UGC namespace (Assets/UGC-Protype/) provides the standard scaffold that every therapy game scene is built on. It encapsulates game lifecycle, parameter reception, visual effects, and level sequencing so game-specific code only needs to implement the actual therapy logic.

UGCGameController

Singleton MonoBehaviour. Owns the game lifecycle: start, pause, resume, stop. Holds references to cameras, timer, level manager, and VFX controller.

UGCParametersController

Receives therapy parameters via Messenger. Maintains a static HashSet<IParam> of all params and dispatches updates to registered IParam instances.

UGCTherapeuticEventsController

Reacts to param changes. Applies visual effects: blur, brightness, contrast, darkness, IPD offset, eye patching (layer), object transparency, and difficulty-driven scaling.

Scene Setup Requirements

Every game scene must have all three components present as GameObjects:

Required scene components
requiredUGCGameControllerOr a game-specific subclass (e.g. UGCSakadyGameController). Must have references to left/right/head cameras assigned in Inspector.
requiredUGCParametersControllerOr a game-specific subclass. Must have scriptableParameter[] array populated with any game-specific params.
requiredUGCTherapeuticEventsControllerOr a game-specific subclass. Must have ColorGradingLwrp reference assigned.
requiredGameTimerReferenced by UGCGameController.gameTimer. Drives session/demo time limit.
optionalUGCLevelManagerReferenced by UGCGameController.levelManager. Required if the game uses the Playable hierarchy for level sequencing. Leave null for games that manage their own level flow.

Timer Setup

Two timer-setup paths run before StartGame():

SetDemoTimer()

Called after instruction finishes (demo mode). Reads LoaderController.StartDemoTimer() which returns the demo_max_time set by DemoPlayButton (default 30s).

Calls gameTimer.SetTimer(demoMaxTime, countDown=true).

SetHomeTherapyTimer()

Called in home mode. Reads game-time parameter from the session-set game's params array (value is in minutes).

Calls gameTimer.SetTimer(minutes × 60f, countDown=false).

Timer only starts (via StartTimerIfDemoOrHomeTherapy()) if gameTimer.currentGameTime > 0. A value of 0 means unlimited time — the game runs until a StopGame or ForceStopGame event arrives from the tablet.

TherapyEvent Receiver

UGCGameController implements ITherapyEventReceiver and subscribes to TherapyEventBroker in Awake(), unsubscribing in OnDestroy().

AcceptedEvents
flagTherapyEventFlags.PauseCalls Pause(true). Sets controller state to EnabledWithoutPointers (shows controllers, hides laser).
flagTherapyEventFlags.ResumeGameCalls Pause(false). Restores controller state to EnabledWithPointers.

Game-specific subclasses can override OnTherapyEvent() and expand AcceptedEvents to handle additional flags.

Playable Hierarchy

The level system is built on a three-tier hierarchy of Playable subclasses. UGCGameController holds a reference to the root UGCLevelManager and calls levelManager.Play(null) to kick off the sequence.

UGCLevelManager

Assets/UGC-Protype/UGCLevelManager.cs

Top-level Playable. Contains an array of named UGCLevelContainer[].

  • BeginPlay() calls PlayContainer(firstContainer) — starts the container named by the firstContainer string field (default "Default")
  • PlayContainer(name) stops any currently running container, finds the named one, and calls container.Play(OnFinishContainer)
  • When the container fires its finish callback, OnFinishContainer() calls Finish() which signals the game is complete via OnFinishAuto → ModuleFinished event
  • PlayContainer() can be called at any time to switch containers mid-game (e.g. difficulty gating)

UGCLevelContainer

Assets/UGC-Protype/UGCLevelContainer.cs

Mid-hierarchy Playable. Groups a set of Playable children (either UGCLevel or nested UGCLevelContainer) and sequences them.

FindNextLevelMethod

Sequentially — plays unfinished levels in index order (1→2→3). Returns null when all are done.

Randomly — picks a random unfinished level. If startFromFirstLevel=true, always starts with index 0 on first play after restart.

Loop Mode

When loop=true the container doesn't finish — it calls Restart() instead of Finish(). Levels are re-spawned from levelPrefabs[] on each restart so they reset to a clean state.

Non-loop mode uses GetComponentsInChildren<UGCLevel> to find levels at startup (no prefab required).

When a child Playable finishes, OnChildPlayableFinished() deactivates it and calls BeginPlay() again to advance to the next. The OnChildPlay UnityEvent fires each time a child starts — useful when level GameObjects don't have direct scene references (since they're spawned from prefabs).

UGCLevel

Assets/UGC-Protype/UGCLevel.cs

Leaf node in the hierarchy. Contains only one public method: FinishLevel() which calls the protected Finish(). Game logic scripts on the same GameObject call GetComponent<UGCLevel>().FinishLevel() when the round is over (correct answer, timeout, etc.).

UGCParametersController

Assets/UGC-Protype/UGCParametersController.cs

Central parameter hub for each game scene. Maintains a static HashSet<IParam> parameters — shared across the entire application lifetime. Params are added to this set either by static field initialisation (the named params below) or by UGCParametersController.AddParam(param).

Why static? Parameters persist across scene loads. This means parameter values carry over from one game to the next without needing a REST round-trip, which reduces first-frame latency. The OnLevelFinishedLoading hook resets difficulty only — other param values remain until explicitly updated by the server.

AwaitFirstParameterArrive()

Returns a UniTask that completes when the first parameter update arrives. UGCGameController.WaitForParamsInit() awaits this before calling Ready(), ensuring the game doesn't start before the server's initial state is applied. Multiple callers can await simultaneously — all are resolved at once via a List<UniTaskCompletionSource>.

Difficulty System

UGCParametersController.difficulty is a static clamped int (0–10) that drives object scale. It is managed separately from the parameter polling system:

  • ComboManager increments/decrements it based on correct/wrong answer events
  • The value is tracked per ObjectSize tier via highestDifficulty[3]
  • When objectSizeParam changes, ResetDifficulty() restores difficulty to the highest previously reached for that size tier (so difficulty isn't lost when size changes)
  • On scene unload, OnLevelFinishedLoading zeroes all difficulty values

The scale multiplier (objectSizeParam.SecondValue) is derived from difficulty in OnDifficultyChange():

ObjectSizeBase scalePer-difficulty decrementRange (diff 0–10)
Small0.7−0.03 / step0.70 → 0.40
Medium1.0−0.03 / step1.00 → 0.70
Big1.3−0.03 / step1.30 → 1.00

UGCTherapeuticEventsController

Assets/UGC-Protype/UGCTherapeuticEventsController.cs

Singleton. Subscribes to IParam.OnUpdate callbacks from every shared param in Start() and unsubscribes in OnDestroy(). Applies the resulting visual effects via ColorGradingLwrp and StandardRP_Blur_Brightness.

Blur / Brightness / Darkness / Contrast

All four colour effects are applied per-eye and respect dominantEyeParam. The dominant eye receives darkening/blur; the non-dominant eye receives brightening. Contrast is applied to the dominant eye only (non-dominant stays at 1).

EffectMethodApplied toRange mapping
BlurApplyBlurDominant(v)Dominant eye only0–100 API → 0–10 shader value
BrightnessApplyBrightness(v)Non-dominant eyeDirect float
DarknessApplyDarkness(v)Dominant eyeStored as negative (0 → −value)
ContrastApplyContrast(v)Dominant eye0–100 API → 1–9 (1=neutral); non-dominant forced to 1

ResetVFX() is called by UGCGameController.OnGameStop() — sets blur=0, brightness=0, contrast=1, darkness=0 before scene transitions.

IPD

SetIPD(int ipd) converts millimetres to metres and offsets the left/right camera localPosition.x symmetrically: 

leftCamera.localPosition.x  = -(ipd / 2f) / 1000f   // e.g. 65mm → left x = -0.0325
rightCamera.localPosition.x = +(ipd / 2f) / 1000f   // right x = +0.0325

Called at Start() with the current default (65mm), then on every currentIPDParam.OnUpdate.

Eye Patching System

Game objects that should appear on a specific eye carry an EyePatchable component. UGCTherapeuticEventsController.SetLayers(param, patchables) iterates all patchables in the given group and calls patchable.Patch(eyeLayer) which recursively sets the Unity layer (LayerMask) on the object and its children up to depth levels deep.

PatchableGroupControlsParamAlternating behaviour
InteractableTarget objects (things to hit/catch)interactableObjectCameraParamAlways alternated; never same layer as Deployer
DeployerSpawner / environment objectsdeployerObjectCameraParamAlways alternated; forced to opposite of Interactable

AlternateEye() is called after a correct answer to swap alternating layers. It calls SwapAlternatingLayer() on each relevant param then re-applies layers to all EyePatchables in the scene.

Transparency

Transparency params (0–100 from API) are remapped to 0.2–0.9 opacity and applied via TransparencyController components grouped by PatchableGroup. Zero from the API maps to full opacity (1f) — meaning "no transparency effect".

Scalable

Assets/UGC-Protype/Scalable.cs

Add to any GameObject that should resize when objectSizeParam changes or when difficulty changes. OnDifficultyChange in the VFX controller calls Scale(objectSizeParam.SecondValue) on every Scalable in the scene.

relativeScale = false (default)

Final scale = Vector3.one × paramScale. All objects with this flag off end up the same absolute size regardless of their starting scale.

relativeScale = true

Final scale = startingScale × paramScale. Objects maintain their proportions relative to each other — useful when different objects should be scaled but stay in ratio.

Scaling is animated by default (scaleGraduallyOverTime=true, scaleTime=0.5s) via a Vector3.Lerp coroutine. The target scale is re-read each frame during the lerp, so mid-animation param changes are handled gracefully. On OnEnable, objects scale in from zero (ScaleFromZero) — giving a pop-in animation each time a level activates.

Extending the Framework

A minimal game-specific implementation requires three subclasses. Here's the typical extension pattern: 

// 1. Game controller — override only what you need
public class UGCSakadyGameController : UGCGameController
{
    public override void StartGame()
    {
        base.StartGame();               // starts levelManager + timer
        SpawnInitialObjects();          // game-specific setup
    }

    protected override void Ready()   // optional — runs before StartGame
    {
        SetupUI();
        base.Ready();
    }
}

// 2. Parameters controller — add game-specific params
public class UGCMiceGameParametersController : UGCParametersController
{
    // Game-specific param declared as new (shadows the base static)
    public new static IntParam numberOfObjectsParam = new IntParam("fruit-count", 3);
}

// 3. VFX controller — override only changed handlers
public class UGCSakadyTherapeuticEventsController : UGCTherapeuticEventsController
{
    protected override void OnObjectSizeChanged(ObjectSize size, float scale)
    {
        base.OnObjectSizeChanged(size, scale); // scales all Scalable objects
        AdjustSpawnRadius(scale);               // game-specific response
    }
}
Adding a game-specific ScriptableParameter: Create the asset via Assets → Parameters → Int/Bool, add it to the game scene's UGCParametersController.scriptableParameter[] array in the Inspector. The parameter is automatically added to the static HashSet in Awake() and removed in OnDestroy(), so it's only active while the game scene is loaded.

Component Reference

UGCGameController

UGC-Protype/UGCGameController.cs

Base singleton. Game lifecycle: Awake → Start → Ready → StartGame. Handles Pause/Resume via TherapyEventBroker. Holds camera refs + GameTimer.

UGCParametersController

UGC-Protype/UGCParametersController.cs

Owns static HashSet<IParam>. Bridges Messenger("paramsUpdated") to typed IParam.UpdateParameter() calls. Manages difficulty reset on scene load.

UGCTherapeuticEventsController

UGC-Protype/UGCTherapeuticEventsController.cs

Reacts to all param OnUpdate events. Applies: IPD, blur, brightness, darkness, contrast, transparency, eye patching, difficulty scaling.

Playable / UGCLevelManager / UGCLevelContainer / UGCLevel

UGC-Protype/Playable.cs et al.

Level hierarchy. Manager holds containers; containers sequence levels (sequential or random, with optional loop). Level calls FinishLevel() when done.

EyePatchable

UGC-Protype/EyePatchable.cs

Marks a GameObject as belonging to a PatchableGroup. On OnEnable applies correct layer. Depth field controls recursion into children.

Scalable

UGC-Protype/Scalable.cs

Reacts to objectSizeParam.SecondValue changes. Animates scale over 0.5s. Supports relative and absolute scaling modes.

IParam / Param<T>

UGC-Protype/IParam.cs, Param.cs

Parameter contract and legacy base classes ([Obsolete]). Concrete types: FloatParam, IntParam, BoolParam, CustomParam, EyeLayerParam.

GameTimer

Assets/GameTimer.cs

Session and demo countdown. SetTimer(seconds, countDown) configures; StartTimer() begins. Fires events on expiry to trigger ModuleFinished.
