Parameter System

How therapy parameters travel from the dashboard to individual game behaviours — covering both the legacy static-param tier and the modern ScriptableParameter tier.

1. Architecture & Data Flow

Parameters are sent by the dashboard REST API, polled by LoaderController roughly every 1.5 s, and broadcast globally via Messenger. UGCParametersController receives the list, finds the matching parameter object by name, and calls UpdateParameter() on it — which parses the raw string value and fires a typed callback so consumers (VFX, game logic) can react.

Dashboard REST API — Sessionsetgame.Param[] list in HTTP response
LoaderController.PollServer() — every ~1.5 s
Messenger.Broadcast("paramsUpdated", List<Param>)

name-match loop over static HashSet<IParam>
IParam.UpdateParameter(Sessionsetgame.Param) — parse string → typed value

Two-Tier Design

The parameter system has two coexisting tiers. The older tier used static C# fields on UGCParametersController; the newer tier uses ScriptableObject assets that can be swapped per-game in the Inspector without touching code.

Tier 1 — Static Params [Obsolete]

Static fields of type IntParam, FloatParam, BoolParam, EyeLayerParam, CustomParam<T>, CustomParam<T,U> declared directly on UGCParametersController.

  • Registered automatically via base Param constructor calling UGCParametersController.AddParam(this)
  • Callback is an Action<T> passed at construction or via OnUpdate +=
  • Shared as public static — any class can read blurParam.Value directly

Tier 2 — ScriptableParameter Preferred

ScriptableObject assets assigned to UGCParametersController.scriptableParameter[] in the Inspector. Each asset carries its own names list, default value, and parse function.

  • Added to the static HashSet<IParam> in Awake()
  • Callback via Changed event — Action<ScriptableParameter<T>, T>
  • Can be replaced per-game by assigning a different asset in the scene
Both tiers share the same IParam interface and live in the same HashSet<IParam>. The matching algorithm in OnParametersUpdated() does not distinguish between them.

When IsAlternatingEye is true, SwapAlternatingLayer() is called by the VFX controller each frame or on a timer to toggle which physical eye renders the object. GetAlternatingLayer() returns the currently active side.

CustomParam<T,U> — Two-Value Params

For parameters that simultaneously control two related values, CustomParam<T,U> (and its base Param<T,U>) stores FirstValue and SecondValue. Examples in use:

  • objectSizeParam — (ObjectSize enum, float distance reference)
  • objectDistanceParam — (DistanceMode enum, float meters)

The parse lambda returns a C# value tuple (T, U) which is destructured directly into the two fields.

3. Tier 2 — ScriptableParameter<T> (Preferred)

Assets/Shared/Scripts/GameParameters/ScriptableParameter.cs

ScriptableParameter<T> is a ScriptableObject that also implements IParam. Because it is a ScriptableObject, each concrete parameter is a project asset — it survives scene changes, can be shared across multiple scenes, and can be replaced per-game by assigning a different asset in the Inspector.

Lifecycle

Scene loads → UGCParametersController.Awake()
foreach scriptableParameter → parameters.Add(param); param.Reset()
Subscriber code calls: myParam.Changed += OnMyParamChanged
Server update arrives → UpdateParameter() called
ParseFunction(startvalue) → newValue (throws on bad input, caught internally)
Changed event fired with (ScriptableParameter<T> source, T newValue)
Scene unloads → UGCParametersController.OnDestroy()
foreach scriptableParameter → parameters.Remove(param)
ScriptableObject.OnDisable() — _onUpdate delegate set to null
Important: ScriptableObject.OnDisable() nulls the internal _onUpdate delegate — this prevents stale cross-scene callbacks. Always subscribe in OnEnable / Start and unsubscribe in OnDisable / OnDestroy to avoid missing the event after a scene reload.

Creating & Assigning a ScriptableParameter

  1. In the Project window: Right-click → Parameters → Bool / Int / BackgroundMode
  2. Name the asset (convention: {Game}_{ParamPurpose}.asset, e.g. MiceGame_FruitCount.asset)
  3. In the Inspector: set Names array to the server-side parameter key(s) and set Default Value
  4. Drag the asset into the game scene's UGCParametersController.scriptableParameter[] array
  5. Subscribe to the Changed event in a MonoBehaviour: myParam.Changed += HandleChange;
Assets live in Assets/THERAPY MODULES/{GameName}/Parameters/. Shared overrides for multiple games live in Assets/THERAPY MODULES/SharedParameters/.

BackgroundModeParameter — Lookup Table

Assets/Shared/Scripts/GameParameters/BackgroundModeParameter.cs

The BackgroundMode struct carries two fields: DisplayOriginalBackground (bool) and BackGroundColor (Color). The asset holds a Pair<string, BackgroundMode>[] lookup table configured in the Inspector, mapping server-sent strings to colour presets. If the incoming string is not found in the table, a ParamParseException is thrown and the value stays unchanged.

// BackgroundMode struct
public static readonly BackgroundMode Default = new BackgroundMode
{
    displayOriginalBackground = true,
    backGroundColor = Color.black
};

// ParseFunction in BackgroundModeParameter
public override Func<string, BackgroundMode> ParseFunction => (stringValue) =>
{
    stringValue = stringValue.ToLower();
    foreach (var pair in NameToColorList)
        if (pair.First == stringValue) return pair.Second;
    throw new ParamParseException(this, $"No background mode found for name {stringValue}");
};

4. Matching & Update Logic

Assets/UGC-Protype/UGCParametersController.cs — OnParametersUpdated()

When the server payload arrives, OnParametersUpdated() iterates over the incoming List<Sessionsetgame.Param> and for each entry performs a linear scan of parameters (HashSet<IParam>) matching param.game_parameters.name against IParam.Name or IParam.SecondName.

foreach (var sessionsetParam in @params)
{
    string paramName = sessionsetParam.game_parameters.name;
    foreach (var ugcParam in parameters)
    {
        if (paramName == ugcParam.Name || paramName == ugcParam.SecondName)
        {
            sessionParam = sessionsetParam;
            param = ugcParam;
            break;
        }
    }
    if (sessionParam != null)
    {
        if (!param.Equals(sessionParam))   // skip if value unchanged
            param.UpdateParameter(sessionParam);
    }
    else
    {
#if UNITY_EDITOR
        Debug.LogWarning($"Unable to find param: {paramName}");
#endif
    }
}

Equality & Skip Logic

Before calling UpdateParameter(), the controller checks !param.Equals(sessionParam). This compares IParam.StringValue (the last raw string received) with sessionParam.startvalue. If they are identical the update is silently skipped, preventing redundant parse and callback overhead on every poll cycle.

Implication for debugging: if a parameter value is sent but the callback never fires, verify that the incoming startvalue string differs from the last received value. You can read IParam.StringValue at runtime via the Editor mock window or Debug.Log.

Error Handling

Parse errors are fully absorbed — they never propagate to the Unity main loop:

  • Tier 1 (Param<T>): catches Exception in UpdateParameter(), calls ParsingFailed() which restores the default value and logs an error including scene name and current level. The OnUpdate callback is not fired.
  • Tier 2 (ScriptableParameter<T>): catches exceptions from ParseFunction and from the _onUpdate delegate separately. Parse failure: logs error, value unchanged. Callback exception: logs error, but StringValue and _value are still updated via finally block.
In Tier 2, if a subscriber's callback throws, the value is still committed (the finally block runs). This means the next poll will see the value as unchanged and skip the update — the subscriber will not get a second chance unless the server sends a different value.

Two exception types are defined in Param.cs:

5. Broker Events Fired by Parameters

Two parameters fire typed Broker<T> events in addition to their own callbacks. This allows scene objects to react without needing a direct reference to the parameter.

Assets/Shared/Scripts/Events/TherapeuticParameterEvents/

ObjectSizeChanged

Fired by objectSizeParam on every size change. Carries ObjectSize enum (Small, Medium, Big).

Subscribe: Broker<ObjectSizeChanged>.Subscribe(this, OnSizeChanged);

[Obsolete] — migrate to ScriptableParameter.

ObjectDistanceChanged

Fired by objectDistanceParam. Carries float Distance (metres) and DistanceMode (Static / Dynamic).

Subscribe: Broker<ObjectDistanceChanged>.Subscribe(this, OnDistanceChanged);

Consumed by AngularSizeable for FOV-correct scaling at varying depths.

The HierarchicalBroker assembly has noEngineReferences: true. Event structs (ObjectSizeChanged, ObjectDistanceChanged) implement IEventArgs from that assembly. They are readonly struct — no heap allocation per event.

6. Async — AwaitFirstParameterArrive()

Assets/UGC-Protype/UGCParametersController.cs — line 312

Game controllers sometimes need to defer game startup until at least one parameter payload has been applied (to avoid running the first frame with all-default values). AwaitFirstParameterArrive() provides a UniTask-based awaitable for this.

// Typical usage in a game controller
await parametersController.AwaitFirstParameterArrive(destroyCancellationToken);
StartGame();   // guaranteed: at least one UpdateParameter() call has completed

Internally it uses a List<UniTaskCompletionSource>. The first call to SignalParameterArrive() (triggered from any UpdateParameter() success) completes all pending tasks and sets a flag so future calls to AwaitFirstParameterArrive() return immediately.

The flag is cleared on scene reload (OnLevelFinishedLoading) so the next game scene must await again. However, the paramsArrived flag is an instance field — it resets when the new scene's UGCParametersController is instantiated.

7. Game-Specific Parameters

Games that require additional parameters subclass UGCParametersController and declare extra static fields. Because the base Param constructor calls UGCParametersController.AddParam(this), game-specific params are automatically registered when the subclass is first touched by Unity's static initializer.

8. How-To Guide

Add a ScriptableParameter (recommended)

  1. Create the asset: Project window → Right-click → Parameters → Bool / Int / BackgroundMode
    Save to Assets/THERAPY MODULES/{GameName}/Parameters/{Name}.asset
  2. Configure in Inspector: set Names (server key, e.g. ["my-param"]) and Default Value.
  3. Assign to scene: drag the asset into UGCParametersController → scriptableParameter[] on the game scene's controller GameObject.
  4. Subscribe in code: 
    // Typically in a MonoBehaviour that handles the effect
[SerializeField] private BoolParameter myParam;

private void OnEnable() => myParam.Changed += OnMyParamChanged;
private void OnDisable() => myParam.Changed -= OnMyParamChanged;

private void OnMyParamChanged(ScriptableParameter<bool> src, bool newVal)
{
    // react to value change
}
  5. Test in Editor: use the [Button] UpdateFromEditor(T newValue) on the asset to simulate a value change without a server connection.

Add a static Param (legacy — avoid for new work)

// In UGCParametersController or a subclass
public static FloatParam myNewParam = new FloatParam("server-key", 0f,
    value => { /* callback */ });

// Read from anywhere:
float v = UGCParametersController.myNewParam.Value;
The base Param constructor calls UGCParametersController.AddParam(this) at field-initialization time. This means the field must be initialized before UGCParametersController.Awake() iterates parameters. Static fields in C# are initialized before any instance constructor runs, so order within the same class is top-to-bottom. Subclass static fields are initialized when the subclass type is first used.

Override a shared param for one game

The simplest way is to shadow the static field with a new declaration in the subclass:

public class UGCMyGameParametersController : UGCParametersController
{
    // Replaces the base objectSizeParam with a game-specific parse function
    public new static CustomParam<ObjectSize, float> objectSizeParam =
        new CustomParam<ObjectSize, float>("object-size-dynamic", ObjectSize.Medium, 1f,
            value => { /* game-specific parse */ });
}

For ScriptableParameter assets: simply assign a different asset of the same type to the scene's scriptableParameter[] array — no code change needed.
